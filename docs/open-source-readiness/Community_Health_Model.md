# Community Health Model (Medical-Adjacent OSS)

**Research Use Only. Not for clinical use.**

## Goals

Because this repository is medical-adjacent, community health is a safety feature:
- reduce misinformation and clinical claims
- respond to vulnerabilities and safety incidents
- keep changes traceable and reviewable

## Minimum Maintainer Expectations

- triage issues weekly
- respond to security reports within 7 days
- publish release notes for user-visible changes
- enforce “no patient data” and “no clinical claims”

## Contribution Pathways

- Issues: bug reports, documentation fixes, reproducibility improvements
- Pull requests: must follow `CONTRIBUTING.md`

## Misuse and Claim Moderation

Maintainers will remove content that:
- contains PHI/PII
- includes clinical efficacy claims
- encourages unsafe self-experimentation

## Sustainability Signals

Track:
- number of active maintainers
- bus factor (minimum 2 maintainers recommended)
- time-to-first-response for issues and PRs
- number of closed security advisories and average time to remediate
